# POS Hardware Service

A production-grade, cross-platform hardware bridge for POS systems, written in Rust.

## Features
- **Cross-Platform**: Windows, Linux, macOS, Android.
- **Protocol**: Secure WebSocket communication.
- **Hardware Support**: Printers (ESC/POS), Cash Drawers, Customer Displays.
- **Architecture**: Asynchronous (Tokio), Modular, Extensible.

## Project Structure
- `src/lib.rs`: Core library logic (portable).
- `src/main.rs`: Desktop binary entry point.
- `src/config`: Configuration handling.
- `src/device_manager`: Manages hardware lifecycles.
- `src/hardware`: Traits and implementations for hardware.
- `src/socket`: WebSocket server implementation.
- `config.toml`: Configuration file.

## Getting Started

1.  **Build**:
    ```bash
    cargo build --release
    ```

2.  **Run**:
    ```bash
    ./target/release/pos_hardware_service
    ```

3.  **Configure**:
    Edit `config.toml` to define your devices.
